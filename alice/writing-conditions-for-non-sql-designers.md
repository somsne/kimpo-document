# 不懂数据库，照样写对筛选条件 —— 给模板设计者的白话指南

> 读者：会用 Excel、会画 Kimpo 模板，但**没学过 SQL、也不想学**的设计者。
> 这篇教你用自己脑子里本来就有的想法，直接写出合法、正确的筛选条件和表达式。全文不出现任何数据库黑话。
> 配套阅读：空值的来龙去脉见 `../fields/empty-values-for-human-developers.md`（想知道"为什么"再看，不看也不影响照做）。

---

## 1. 一句话心法：把条件念出来

每一条筛选条件，都是一句完整的中国话：

> **谁的什么 + 怎么样 + 什么值**

- "**采购订单的订单编号** *等于* **本报表的订单编号**"
- "**销售订单的订单日期** *不早于* **本报表的开始日期**"
- "**人员信息的岗位** *包含* **本报表填的关键字**"

写条件前先把这句话念顺。念不顺的条件，写出来也是错的。

三种"值"你要分清（界面上选的时候就是三个来源）：

| 你想用的值 | 在编辑器里 | 例子 |
|---|---|---|
| **别的表里的数据** | 从来源表选字段 | 采购订单的订单编号 |
| **本报表上填的内容** | 从"本报表"选字段 | 查询页上用户填的开始日期 |
| **写死的固定值** | 直接输入 | `'已审核'`、`100` |

另外有几个**系统变量**随取随用：当前用户、当前用户所在部门、当前时间、今天。"只看我自己的单子"就是一句：`制单人 等于 系统变量.当前用户`。

## 2. 配方库：最常用的七种条件，照抄就行

### 配方① 精确匹配（对单据、对编号）

> 想法："把和这张入库单同一个订单号的采购明细找出来。"

```
[采购订单_主表].[订单编号] 等于 本报表.[订单编号]
```

没填订单编号时会怎样？**什么都查不到**——没有编号当然配不上任何单子，这正是你想要的（不会莫名把全库拉进来）。

### 配方② 日期/数量区间（开始、截止随便填不填）

> 想法："只看开始日期到截止日期之间的单子；没填就是不限。"

```
[销售订单_主表].[订单日期] 不早于(≥) 本报表.[开始日期]
并且 [销售订单_主表].[订单日期] 不晚于(≤) 本报表.[截止日期]
```

**没填的那头自动不限**：都没填=查全部；只填开始=只管下限；只填截止=只管上限。你**不需要**像老系统那样在每条后面补一句"或者 开始日期=''"——当然，写了也不会错。数量、金额区间同理（`数量 ≥ 本报表.最小数量`）。

### 配方③ 模糊搜索（搜索框式）

> 想法："名称里带这个词的都列出来；搜索框空着就全列。"

```
[指标库].[指标名称] 包含 本报表.[关键字]
```

关键字没填=全部列出（和所有搜索框一个习惯）。还有「开头是」「结尾是」可选。

### 配方④ 多个可选筛选器组合（下拉框选了才过滤）

> 想法："岗位、部门、学历三个下拉，选了哪个就按哪个过滤，没选的不管。"

```
([人员信息].[岗位] 等于 本报表.[岗位]     或者 本报表.[岗位] 为空)
并且 ([人员信息].[部门] 等于 本报表.[部门] 或者 本报表.[部门] 为空)
并且 ([人员信息].[学历] 等于 本报表.[学历] 或者 本报表.[学历] 为空)
```

念出来就是："岗位等于所选，**或者压根没选岗位**。"这里的「为空」就是"没填/没选"的意思。老系统写 `=''` 的，原样写也认。

### 配方⑤ 状态门槛（固定值过滤）

> 想法："只要待评审的。"

```
[需求上报_主表].[评审状态] 等于 '待评审'
```

### 配方⑥ 必填门闸（没填就什么都不给）

> 想法："必须先填了订单编号，才允许带出数据。"

```
[采购订单_主表].[订单编号] 等于 本报表.[订单编号]
并且 本报表.[订单编号] 不为空
```

第二句是"门闸"：编号没填时整个查询关门。配方①本来就"没填查不到"，加这句是把意图写明白，永远不会失效。

### 配方⑦ 排除（不看某些的）

> 想法："别把这一单自己算进去。"

```
[入库单_扩展表].[明细号] 不等于 本报表.[明细号]
```

本报表明细号没填时？**什么都不排**（没指定要排谁，当然谁都不排）。

## 3. "没填"到底会发生什么：一张表背下来

参数（本报表的字段、系统变量）**没填**时，各种条件的固定行为：

| 条件长什么样 | 参数没填时 | 一句话理由 |
|---|---|---|
| `… ≥ / ≤ / > / < 参数`（区间） | 这条**当没写** | 没提要求就不限 |
| `… 包含/开头是/结尾是 参数` | 这条**当没写** | 搜索框空=不搜 |
| `… 不等于 参数` | 这条**当没写** | 没说排谁就不排 |
| `… 等于 参数` | **什么都配不上** | 没有编号配不上任何单 |
| `… 或者 参数 为空` | 强制"不限" | 你亲口说了没填也要 |
| `… 并且 参数 不为空` | 强制"关门" | 你亲口说了没填免谈 |

记不住就记两头：**"等于"最严格，其他都宽容；想改行为就多写半句（为空/不为空）。**

## 4. 表达式（算值）的三条白话规则

在输出列、取数表达式里算值时：

1. **没填的数不会替你当 0**。`数量 × 单价`，数量没填结果就是空——"不知道乘出来当然不知道"。想按 0 算，用「空值默认」函数写明白：`空值默认(数量, 0) × 单价`。
2. **拼文字时没填的部分自动跳过**。`拼接("编号-", 编码)`，编码没填得 `"编号-"`，不会整串消失。
3. **日期没填就别指望它算出日期**。`加天数(交货日期, 7)`，交货日期空结果就是空——绝不会冒出个 1900 年。

汇总数字时：合计把空的跳过（全空=0）；平均数不把没填的算进人头；"计数"只数填了的。**这些都是系统自动的、正确的，不用你处理。**

## 5. 常见错误 → 正确写法

| 你可能想写 | 问题 | 正确写法 |
|---|---|---|
| `数量 等于 0` 来找"没填数量的" | 0 是填了的零，不是没填 | `数量 为空` |
| `日期 等于 '1900-01-01'` 找"没填日期的" | 新系统里没有 1900 占位 | `日期 为空` |
| 三个下拉筛选全用"或者"连 | "或者"=满足任意一个就要，越选越多 | 不同字段之间用"**并且**"，每个字段内部再"或者 为空"（配方④） |
| 区间写反：`订单日期 ≤ 开始日期` | 念一遍就发现不对："订单日期不晚于开始日期"？ | `订单日期 ≥ 开始日期`（开始是下限） |
| 想"没选就全部"，于是不敢用等于 | 等于+没填=查不到，这是保护你 | 要"没选就全部"就加 `或者 参数 为空` |
| 在"包含"里自己拼百分号 `'%'+词+'%'` | 不需要，「包含」自带 | `字段 包含 词` |

## 6. 黄金三步：写完必做

1. **念一遍**：每条条件念成人话，念不顺=写错了。
2. **点预览**：编辑器的"校验/预览"会告诉你——语法对不对、**哪些条件因为参数没填而没有生效**（"未生效条件"清单）。查出来的数比预期多，八成是某条被判"当没写"，看清单就明白。
3. **空着试一遍**：把可选参数全留空跑一次（应该查出全部/或什么都没有——按你的设计），再各填一个跑一次。两遍都对，才算写完。
