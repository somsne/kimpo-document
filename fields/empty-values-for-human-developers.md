# 字段空值：Kimpo 为什么"空就是空"，而不是填 0 —— 面向人类的讲解

> 读者：模板设计者、插件开发者，尤其是**用惯了同类产品的老手**。
> 读完你会明白：空、0、空字符串到底是几个概念；Kimpo 的空值规则在公式、表达式、筛选三个地方各是什么样；以及你原来的习惯要怎么平移——绝大多数场景一个字都不用改。
> 配套：查询/筛选的完整用法见 `../alice/`（含零 SQL 基础的白话指南）；插件读写契约见 `../aria/`。

---

## 1. 先把概念掰清楚：这几个东西不是一回事

| 写法 | 它是什么 | 在 Kimpo 里 |
|---|---|---|
| **空**（界面语言：为空 / 没填；系统内部记法：null） | "**这一格没有被填过**"这个事实本身 | 真实存在的状态。数据库、数据权威层（Mirror，代号 艾莉亚/Aria）、接口里都如实保留 |
| **0** | 一个**真实的数值**——用户亲手填的零 | 有值。库存 0 件 ≠ 库存没盘点 |
| **""（空字符串）** | 一个"内容为零个字符"的文本 | **在 Kimpo 里不存在**。填入纯空白会被归一成"空"；表达式里写 `=''` 会被自动理解为"为空"——所以你不需要、也无法区分 "" 和空 |
| **false（勾选框没勾）** | 勾选框的"否" | 勾选框**没有空这一态**：没勾就是否。需要"未表态"就换单选字段 |
| **1900-01-01** | 老系统给空日期垫的哨兵 | **已废止**。空日期就是空，永远不会冒出 1900 年 |

一句话：**"没填"和"填了个零/零个字符"是不同的业务事实，Kimpo 把它们分开记；老模式把它们混在一起记，混掉的信息永远找不回来。**

- null、空、没填、为空——**这四个词说的是同一个东西**，只是场合不同：界面和文档说"空/没填"，数据库和开发者说 null。用户永远不会在界面上看到 null 这个词。
- 需要记的对立面只有一组：**空 ≠ 0，空 ≠ ""（后者干脆不存在了）**。

## 2. 两种方案：分歧在哪

老模式（同类产品普遍做法）：没填就偷偷垫一个"无害值"——字符串 `""`、数值 `0`、日期 `1900-01-01`。好处是全系统不需要"空"这个概念；代价是"没填"这个事实在落库瞬间被永久抹掉。

Kimpo 模式：没填就如实记"空"，同时给公式、表达式、筛选各配一套**确定的空值规则**（下一节），让空值在使用时自动表现得合乎直觉。

> 老模式把"方便计算"提前焊死在**存储**里；Kimpo 把它推迟到**计算的那一瞬间**。数据只记录事实，方便是引擎的责任。

### 一个故事看懂差别：仓库盘点

三个仓库上报库存：A 库报 120，B 库报 **0**（真空了），C 库**忘了填**。

老模式下 C 被垫成 0：平均库存 = (120+0+0)÷3 = **40**（被假 0 拉低）；"库存为 0"筛出 B 和 C，但谁是真零谁是没盘**永远无法回答**；"哪些仓库没交盘点"这个问题在数据里已经不存在了。

Kimpo 模式下 C 是空：平均 = (120+0)÷2 = **60**（空不进分母）；"库存为 0"只有 B，"库存**为空**"只有 C——两个问题两个答案都对；公式 `=库存 × 单价` 对 C 照样算出 0，日常使用毫无障碍。

日期版本更扎眼：老模式没填的"合同签订日"= 1900-01-01，它会被打印、排序排最前、掉进"早于 2000 年"的筛选。每个用过老模式的实施顾问都清洗过这种"上个世纪的合同"。

## 3. 空值规则总览：三个地方，三套确定行为

Kimpo 里你会在三个地方跟"式子"打交道，空值在三处的表现**刻意不同**，各自对齐你在那个场合的直觉：

| 场合 | 世界观 | 空的表现 |
|---|---|---|
| **单元格公式**（模板格子里的 `=数量*单价`） | Excel 世界 | 空自动扮演角色：算术当 0、拼文本当空文本、判断当"否" |
| **取数/动作表达式**（计算列、写值、触发条件） | 数据库世界 | 空诚实传播：空进空出，不替你捏造 0 |
| **筛选条件** | 固定规则 | 分两种"空"：**数据的空**从严，**参数的空**=没提要求 |

### 3.1 单元格公式：像 Excel 一样

| 场景 | 结果 | 例子 |
|---|---|---|
| 算术 | 空当 **0** | 数量空：`=数量 × 单价` → 0 |
| 文本函数/拼接 | 空当**空文本** | `=LEN(备注)` → 0；`="编号-" & 数量` → `"编号-"`（不是 `"编号-0"`） |
| 逻辑判断 | 空当**否** | `=IF(是否加急,"急","普通")` 未勾 → "普通" |
| 日期函数 | **结果为空**（不报错、不当 1900） | `=DATEDIF(签订日, 今天)` 签订日空 → 空 |
| 显式判空 | `ISBLANK(x)` | `=IF(ISBLANK(库存), "未盘点", 库存)`；配套还有 ISNUMBER/ISTEXT/ISERROR |

### 3.2 取数/动作表达式：数据世界不说谎

| 规则 | 例子 |
|---|---|
| 算术空传播 | `数量 × 单价`，数量空 → **空**（未知乘任何数还是未知） |
| 日期空传播 | `加天数(交货日期, 7)`，交货日期空 → 空 |
| **唯一例外：拼接跳空** | `拼接("编号-", 编码)`，编码空 → `"编号-"`（整串不会消失） |
| 要"空当 0"就写明白 | `空值默认(数量, 0) × 单价`（函数面板叫「空值默认」，老名 ISNULL/IFNULL 都认） |

为什么这里不学 Excel？因为表达式的产物是**数据**（要写回字段、进报表）。管道中段替你捏一个 0 写进账本，等于把老模式的哨兵从后门放回来——"运费 + 货款(未知)" 的诚实答案是"未知"，不是"= 运费"。想兜底，`空值默认` 写出来，人人看得见。

### 3.3 筛选条件：分清"数据的空"和"参数的空"

一条筛选条件两边的"空"含义不同：

- **数据列**（被筛的表的字段，如 `制单日期`）为空 = "这单没填"——是**事实**，从严：不会被任何比较命中（连"不等于"也不命中），找它只有「为空」一个入口。
- **参数**（本报表的字段、系统变量、查询输入，如 `开始日期`）为空 = "**用户没提这个要求**"——按固定规则处理：

| 条件类型 | 参数没填时 | 直觉 |
|---|---|---|
| 区间：`≥ ≤ > <`（含 datediff 写法） | 条件**当没写**（自动不限） | 没提要求就不限 |
| 模糊：包含 / 开头是 / 结尾是 | 当没写 | 搜索框空 = 不搜 |
| 排除：`≠` | 当没写 | 没说排谁就不排 |
| 等值：`=` | **保留，恒配不上** | 没有单号当然配不上任何单 |

三个要点：

1. **没有任何开关要你设置**——规则固定，且怎么写都认：参数在左在右、正写反写、套函数（`制单日期 ≥ 开始日期往前7天`）都一样；"并且/或者"任意组合都正确（"或者"组里只当没写那一条，不殃及兄弟条件）。
2. **想反着来，用你原来就会的写法**：要"没填也查全部"补 `或者 参数为空`；要"没填一条都不给"补 `并且 参数不为空`。这两句**永远压过固定规则**。老写法 `或者 参数=''`、`并且 参数<>''` **原样可用**（`=''` 被自动理解为"为空"）。用「空值默认」显式兜底的条件也绝不会被裁。
3. **被"当没写"的条件看得见**：设计态预览会列出"本次未生效条件"清单——查出来的数比预期多，先看它。

### 3.4 统计、排序、分组、匹配

| 场景 | 空的行为 |
|---|---|
| 求和 SUM | 跳过空；全空 = 0 |
| 平均 AVG | 跳过空（**不进分母**）；全空 = 空 |
| 最大/最小 | 跳过空；全空 = 空 |
| 计数 COUNT(字段) | 不计空；行数 COUNT(*) 照计 |
| 排序 | 空值**恒排最后**（升降序都是），不会开屏先见一堆空行 |
| 分组 | 没填的归成一个「(空白)」组，一眼看到"还有多少没填" |
| 取数按行列匹配 | 匹配键任一段为空的行**不参与匹配**（空 ≠ 空）。"没填也算一档"的业务给字段配默认值，别指望空键互配 |
| 展示/导出/打印 | 空一律显示空白；导出 Excel 就是空单元格 |

### 3.5 各类型的"空"长什么样

| 类型 | 什么算空 | 特别说明 |
|---|---|---|
| 文本 | 没填，或只输入了空格 | 全空白输入归一为空；**不存在**"内容是空字符串"的状态 |
| 数值 | 没填 | 显式输入 `0` 是**有值** |
| 日期/时间 | 没选 | 永不变成 1900-01-01 |
| 勾选（布尔） | **没有空** | 没勾 = 否 |
| 附件 | 没有上传任何文件 | — |
| 编号（发号器） | 尚未发号 | 发号后恒有值 |

## 4. 现方案的优点与缺点（诚实版）

**优点**：①信息不丢失（"填了 0"和"没填"永远可区分、可考核）②统计天然正确（不用手工剔假 0）③报表干净（没有 1900 和满屏 0）④必填校验有可靠判据⑤与数据库/Excel/外部 API 的标准语义零翻译接轨。

**缺点（不回避）**：①多一个概念——设计复杂模板时偶尔要意识到"空 ≠ 0"（日常公式无感，显式判空才用 ISBLANK）②"不等于"不含空要适应一下（这是数据库和 Excel 筛选共同的标准行为）③依赖"日期恒有值"的老公式需要检查④插件开发者要处理 null（SDK 的 TypedValue 已把它显式化，照 aria 契约写即可）。

## 5. 老用户快速适应指南

| 你的老习惯 | 在 Kimpo 里 | 要改吗 |
|---|---|---|
| 直接写 `=数量 × 单价`，没填也当 0 | 单元格公式行为完全一样 | ❌ 不用改 |
| `=IF(金额 > 1000, …)` 之类的公式比较 | 行为一样（公式里空数值比较当 0） | ❌ 不用改 |
| 拼接、LEN、文本函数随便用 | 行为一样且更好（空拼出来是空，不是 `"0"`） | ❌ 不用改 |
| 靠"空日期=1900"让日期区间没填时"不限"（直接比较或 datediff 写法） | 固定规则自动不限：都没填查全部、只填开始限下界、只填截止限上界 | ❌ 不用改 |
| 每条筛选手写 `或者 参数=''` / `并且 参数<>''` | **原样可用**且永远压过固定规则；也可以省掉样板 | ❌ 一个字都不用改 |
| `形如 '%参数%'`、`charindex(参数,列)>0` 模糊搜索 | 改用「包含」，行为一致（没填 = 不限） | ✅ 换写法，行为不变 |
| `isnull(列, 0)` 给取数结果兜底 | 原名可用（=「空值默认」） | ❌ 不用改 |
| 用 `=IF(日期="1900-1-1","未填",…)` 判断日期没填 | 改 `=IF(ISBLANK(日期),"未填",…)` | ✅ 改一处 |
| 用 `数量 = 0` 判断"没填" | 真问"等于零"就不动；问"没填"用 `ISBLANK(数量)`（公式）或「为空」（筛选）——**老模式逼你混为一谈的两个问题，现在可以分开问了** | ✅ 按意图改 |
| 报表手工剔除假 0 再算平均 | 不用剔了，平均数天然正确 | ❌ 删掉老补丁 |
| 给日期字段垫"很早的默认值"占位 | 别垫。真要预填就配字段默认值（如默认今天） | ✅ 放弃占位习惯 |

三条心法：

1. **公式里，像 Excel 一样**——不确定时想想 Excel 空单元格会怎样。
2. **筛选里，分清数据和参数**——数据的空是事实（从严），参数的空是没提要求（不限）；想反着来就多写半句「为空/不为空」。
3. **表达式里，空进空出**——要兜底就写「空值默认」，让意图人人看得见。

## 6. FAQ

**Q：用户根本不懂 null，会不会被吓到？**
不会，用户永远看不到 null 这个词。没填显示空白、筛选器里是「为空」、公式里空自动扮演角色。null 只是账本内部记法。

**Q：为什么"不等于 100"不包含没填的单据？**
"没填"= 不知道是多少；不知道的东西既不能说等于 100，也不能说不等于 100。想连没填的一起要，加"或 金额为空"。

**Q：勾选框为什么没有空？**
勾选框在用户眼里只有勾/不勾两种样子，硬造第三态只会让公式和筛选平添分支。业务上真需要"未表态"，用单选字段（是/否/未表态）。

**Q：我想让没填的数量存为 0（比如要导给不认空值的老系统）？**
给字段配**默认值 0**——让 0 成为用户看得见、改得了的真值，而不是系统偷偷垫的。这是"垫值"唯一正当的姿势。

**Q：区间筛选写反了方向怎么办？**
把条件念成人话就能发现："订单日期不晚于开始日期"显然不对。写完条件先念一遍，再用预览验证——预览还会告诉你哪些条件因参数没填而未生效。
