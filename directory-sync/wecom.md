# 企业微信通讯录同步 · 手把手教程

> 把企业微信里的部门和员工同步进 Kimpo。
> 建议先读[总览](./README.md)了解「认领 / 权威模式 / 工号 / 密钥安全」这几个概念。

本教程分两大段：**A. 在企业微信侧准备** → **B. 在 Kimpo 控制台配置**。

---

## ⚠️ 先说一个关键选择（很重要）

企业微信读通讯录有两条路，别选错：

- **✅ 自建应用（推荐，本教程用这个）**：一个"可见范围=全部"的自建应用，能读到员工的**完整信息**（姓名、部门、手机等）。
- **❌ 通讯录同步助手**：从 2022 年起，用"通讯录同步"secret 调接口时，**只返回 ID（userid、部门 ID），不再返回姓名/手机/部门名等**（官方为保护隐私的调整）。用它同步进来的人会**没有姓名**，不适合我们的场景。

所以下面走**自建应用**这条路。

---

## A. 在企业微信侧准备

你需要一个"自建应用"、把它的可见范围设为全部、并拿到 **企业 ID（CorpID）** 和 **应用的 Secret**。

> ⚠️ 你必须是这个企业微信的**管理员**。

### A1. 创建自建应用

1. 管理员登录企业微信管理后台 👉 https://work.weixin.qq.com/
2. 顶部进入 **【应用管理】** → 下方 **「自建」** 区域 → **【创建应用】**。
3. 上传 logo、填应用名称（如"Kimpo 通讯录同步"），**可见范围**选 **全部员工/全部部门**（这是能读到全公司通讯录的关键），创建。

### A2. 拿 CorpID 和 应用 Secret

- **CorpID（企业 ID）**：管理后台 → **【我的企业】** → **企业信息** → **企业ID**。
- **应用 Secret**：进入刚建的应用详情页，找到 **Secret**，点【查看】/【获取】（可能会发到你的企业微信）。同页还有个 AgentId，本同步用不到。

### A3. 配置企业可信 IP

企业微信要求：服务端调用通讯录接口的机器 IP，必须在应用的**可信 IP** 里。

- 在应用详情页找到 **「企业可信IP」/「网页授权及 JS-SDK」** 附近的可信 IP 配置；
- 让部署 Kimpo 的运维同事，把 Kimpo 服务器的**公网出口 IP** 填进去。
- 否则调用会被拒（提示 IP 不在白名单）。

### ✅ A 段产出

| 项 | 从哪拿 | 敏感？ |
|---|---|---|
| 企业 ID（CorpID） | 我的企业 → 企业信息 | 否，可填界面 |
| 应用 Secret | 应用详情页 → Secret | ✅ **敏感，走环境变量** |

> 说明：企业微信这一家，Kimpo 目录源里**只用到 CorpID 和 Secret 两项**（不需要填应用 ID/client_id）。

---

## B. 在 Kimpo 控制台配置

### B0. 密钥交给运维设环境变量

把 A 段的 **应用 Secret** 交给部署同事，设成 Kimpo 服务器的环境变量，例如：

```
WECOM_APP_SECRET = <你的应用 Secret>
```

记住这个**变量名**（`WECOM_APP_SECRET`），界面里只填名字、不填密钥本身。详见[总览 §5](./README.md#5-一条安全铁律密钥不落库不填明文)。

### B1. 新建目录源

**控制台 → 外部目录** → **【新建目录源】**：

| 表单项 | 填什么 |
|---|---|
| **目录类型** | 选「企业微信」 |
| **名称** | 如"总部企业微信" |
| **企业 ID（corp_id）** | 填 A 段的 CorpID |
| **应用 ID（client_id）** | 企业微信不用，留空即可 |
| **凭据引用** | 填环境变量名 `WECOM_APP_SECRET` |
| **权威模式** | 「仅补全本地空字段」（推荐） |
| **匹配策略** | 「自动绑定已有对象」（开启认领） |
| **登录来源码** | 默认 `wecom` |
| **锚点部门** | 一般留空 |

保存。

### B2~B5. 测试连接 → 预览 → 认领 → 应用

和钉钉完全一样，见[钉钉教程 B2~B5](./dingtalk.md#b2-测试连接) 的操作说明：

1. **测试连接** → 显示"连接成功：部门 X · 用户 Y"。
2. **同步预览** → 生成差异（不写数据）。
3. **差异审阅** → 需要合并的手动【绑定到既有部门/用户】认领。
4. **应用已选** → 真正写入。

### B6. 关于工号

企业微信**没有标准的工号字段**，所以同步进来的员工，Kimpo 会自动生成系统工号（形如 `U000123`），你以后可以在部门用户里改成真实工号。企业微信的 userid 只作登录身份，不当工号。

## 常见问题

| 现象 | 原因 & 解决 |
|---|---|
| 同步进来的人没有姓名 | 你用了"通讯录同步助手"secret（只回 ID）。改用**自建应用**的 Secret（见本文开头的关键选择） |
| 调接口报 IP 不可信 | 可信 IP 没配。让运维把 Kimpo 服务器公网出口 IP 加进应用的企业可信 IP |
| 测试连接报读不到部门/成员 | 应用**可见范围**不是"全部"。回 A1 把可见范围改成全部员工/全部部门 |
| 报 secret 无效 | Secret 填错或环境变量没设对；核对变量名与"凭据引用"一致 |
