# 飞书通讯录同步 · 手把手教程

> 把飞书里的部门和员工同步进 Kimpo。
> 建议先读[总览](./README.md)了解「认领 / 权威模式 / 工号 / 密钥安全」这几个概念。

本教程分两大段：**A. 在飞书侧准备** → **B. 在 Kimpo 控制台配置**。

---

## A. 在飞书侧准备（约 5 分钟）

你需要一个"飞书企业自建应用"，给它开通读通讯录权限，然后拿到 **App ID** 和 **App Secret**。

> ⚠️ 你需要是这个飞书企业的**管理员**（或有创建应用的权限）。个人也可以免费创建一个飞书团队。

### A1. 创建企业自建应用

1. 登录飞书开放平台 👉 https://open.feishu.cn/
2. 进入 **开发者后台** → 点 **【创建应用】** → 选 **「企业自建应用」**。
3. 填应用名称（如"Kimpo 通讯录同步"）、图标、描述，创建。

### A2. 拿 App ID / App Secret

进入应用详情页，左侧点 **【凭证与基础信息】**，在"应用凭证"区域复制：

- **App ID**（形如 `cli_xxxxxxxx`）
- **App Secret**（一长串）

### A3. 开通"读通讯录"权限

1. 应用详情页左侧点 **【权限管理】**。
2. 搜索/找到 **通讯录** 相关的**只读**权限，勾选这些（可用"批量导入"一次加）：
   - `contact:department.base:readonly` —— 获取部门基础信息
   - `contact:user.base:readonly` —— 获取用户基础信息
   - （如需读组织架构树/成员归属，一并勾选通讯录里对应的只读权限）
3. **配置通讯录数据范围**：飞书还需要设置应用"能看到哪些部门/成员"。在应用的 **数据权限 / 通讯录权限范围** 里，设为 **全部成员**（或你要同步的范围）。这一步可能需要企业管理员在管理后台授权。

### A4. 发布版本让权限生效

飞书的权限要**发布一个版本**才生效：

1. 应用详情页 → **【版本管理与发布】** → **创建版本** → 填版本号和说明 → **申请发布**。
2. 企业自建应用通常由**管理员审核通过**后即生效（你若是管理员，去管理后台通过即可）。

### ✅ A 段产出

| 项 | 从哪拿 | 敏感？ |
|---|---|---|
| App ID | 凭证与基础信息 | 否，可填界面 |
| App Secret | 凭证与基础信息 | ✅ **敏感，走环境变量** |

> 说明：飞书这一家，Kimpo 目录源用 **应用 ID（client_id）= App ID** 和 **凭据 = App Secret**，**不需要填企业 ID**。

## B. 在 Kimpo 控制台配置

### B0. 密钥交给运维设环境变量

把 A 段的 **App Secret** 交给部署同事，设成环境变量，例如：

```
FEISHU_APP_SECRET = <你的 App Secret>
```

记住变量名 `FEISHU_APP_SECRET`，界面只填名字、不填密钥本身。详见[总览 §5](./README.md#5-一条安全铁律密钥不落库不填明文)。

### B1. 新建目录源

**控制台 → 外部目录** → **【新建目录源】**：

| 表单项 | 填什么 |
|---|---|
| **目录类型** | 选「飞书」 |
| **名称** | 如"总部飞书" |
| **企业 ID（corp_id）** | 飞书不用，留空即可 |
| **应用 ID（client_id）** | 填 A 段的 **App ID** |
| **凭据引用** | 填环境变量名 `FEISHU_APP_SECRET` |
| **权威模式** | 「仅补全本地空字段」（推荐） |
| **匹配策略** | 「自动绑定已有对象」（开启认领） |
| **登录来源码** | 默认 `feishu` |
| **锚点部门** | 一般留空 |

保存。

### B2~B5. 测试连接 → 预览 → 认领 → 应用

和钉钉完全一样，见[钉钉教程 B2~B5](./dingtalk.md#b2-测试连接)：

1. **测试连接** → "连接成功：部门 X · 用户 Y"。
2. **同步预览** → 生成差异（不写数据）。
3. **差异审阅** → 需要合并的手动认领【绑定到既有部门/用户】。
4. **应用已选** → 真正写入。

### B6. 关于工号

飞书用户如果填了**员工编号**（飞书的 `employee_no` 字段），Kimpo 的工号就用它；没填则自动生成系统工号 `U000123`。飞书的 open_id/user_id 只作登录身份，不当工号。

## 常见问题

| 现象 | 原因 & 解决 |
|---|---|
| 测试连接读不到部门/成员 | 通讯录数据范围没设成"全部"，或版本没发布。回 A3/A4 检查 |
| 报权限不足 / scope 缺失 | 通讯录只读权限没勾全或没发布版本。回 A3 补齐、A4 重新发版 |
| 报 app_secret 无效 | Secret 填错或环境变量没设对；核对变量名与"凭据引用"一致 |
| App ID 填哪个字段 | 飞书的 App ID 填 Kimpo 的「应用 ID（client_id）」，企业 ID 留空 |
